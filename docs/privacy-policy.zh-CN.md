# IChat 隐私政策

最后更新：2026 年 3 月 27 日

本隐私政策用于说明 IChat 在你使用扩展时如何处理相关信息。

语言：
[English](./privacy-policy.md) | [简体中文](./privacy-policy.zh-CN.md)

## 概述

IChat 是一个 Chrome 扩展。它只会在你明确触发时抓取当前网页的上下文，并将整理后的提示发送到你在扩展设置中选择的 AI 提供方。

IChat 当前采用本地优先、BYOK（自带密钥）的产品架构：

- 你的模型提供方 API Key 存储在浏览器扩展的本地存储中
- 抓取到的上下文和本地聊天状态保存在你的设备上
- 当前实现中没有项目自建后端
- IChat 会直接把请求发送到你选择的模型提供方

## IChat 处理的信息类型

根据你的使用方式，IChat 可能处理以下几类信息。

### 1. 提供方配置

保存在扩展本地存储中的信息包括：

- OpenAI-compatible API Key
- Gemini API Key
- Anthropic API Key
- 当前选中的提供方和模型设置
- 可选的 OpenAI-compatible 自定义 endpoint

### 2. 页面上下文

当你明确触发抓取时，IChat 可能收集：

- 当前页面的 URL、标题和 host
- 选中的文本
- 智能 DOM 目标文本
- 周边隐式上下文文本
- XPath、CSS path 等定位信息
- 视口和文档元数据

### 3. 抓取到的媒体内容

如果当前目标包含图片，IChat 可能处理：

- 图片源 URL（在可用时）
- 归一化后的图片 blob，并保存在本地
- 图片相关元数据，例如类型、尺寸、alt 文本、caption 文本和 nearby text

如果无法直接解析目标媒体，IChat 可能会为完成本次抓取而使用截图回退，对当前可见区域的目标部分进行截图处理。

### 4. 本地聊天状态

保存在本地的信息还包括：

- 按提供方区分的聊天线程快照
- pending prompt
- capture status 和 dispatch status

## 何时收集数据

IChat 不会为了远程处理而持续监控你的全部浏览活动。

IChat 只有在你明确触发扩展行为时才会处理页面上下文，例如：

- 点击扩展 action
- 使用配置好的抓取快捷键
- 发送带有已抓取上下文的消息

## 数据存储位置

在当前实现中，IChat 主要将数据保存在浏览器本地：

- 扩展本地存储中保存设置、API Key、FlowContext、pending prompt 和聊天快照
- IndexedDB 中保存本地图片附件

当前版本没有云同步，也没有项目自建账号系统。

## 数据会发送到哪里

当你发送请求时，IChat 可能会把相关内容发送给你在设置中选择的提供方，例如：

- 组合后的 FlowContext prompt
- 请求中附带的最近对话消息
- 视觉模型流程需要的图片附件

根据你的配置，请求可能直接发往：

- OpenAI
- Google Gemini
- Anthropic
- 你自行配置的其他 OpenAI-compatible 提供方

这些第三方提供方会按照它们各自的条款和隐私政策处理请求。

## IChat 当前不会做的事情

在当前实现中，IChat 不会：

- 把你的数据发送到项目自建后端
- 要求你注册项目自建账号
- 内置分析、广告或遥测服务
- 出售个人信息

## 用户控制

你可以自行决定是否使用 IChat，以及发送哪些内容。

当前可用的控制方式包括：

- 决定何时触发抓取
- 选择使用哪个模型提供方
- 编辑模型和提供方设置
- 在本地清除某个提供方的聊天线程
- 在本地重置当前抓取到的上下文
- 在发送前移除图片附件

如果未来增加更完整的数据删除或重置功能，本政策也应同步更新。

## 数据保留

保存在本地的数据会一直留在你的设备上，直到你修改、覆盖、删除，或通过浏览器层面的扩展数据移除机制将其清除。

第三方模型提供方的保留策略由你选择的提供方决定，不由 IChat 控制。

## 儿童

IChat 不面向 13 岁以下儿童。

## 安全

IChat 使用浏览器扩展本地存储来保存设置和状态。但任何浏览器扩展都无法保证绝对安全。如果你处理的是高度敏感信息，应先理解本地存储和第三方 AI 提供方处理带来的风险，再决定是否使用 IChat。

## 政策更新

如果产品行为发生变化，尤其是存储、权限、联网方式或数据流发生变化时，本隐私政策也应同步更新。

## 联系方式

当前项目尚未公开一个专门的支持联系方式。等公开仓库和项目联系渠道确定后，本节应更新为稳定可用的联系方式。
